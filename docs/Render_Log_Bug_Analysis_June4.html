3. Bug Details

BUG-1 P0 — phone_collected=True Without Real Phone

Log: SESSION_STATE_SYNCED phone_collected=False → LEAD_SIGNALS → phone_collected=True
User text: "Net jodnu chha kya mero gharama" (no phone number)

Root cause: Telegram user_id is stored as contact_value, then treated as phone.

// lead_extractor.py — detect_contact_method()
if channel == "telegram" and user_id:
    return "telegram", str(user_id)   // ← 2071979418 stored as contact

// session_state_manager.py — sync_session_state()
phone = lead.contact_value   // ← telegram ID treated as phone
if phone:
    state.phone_collected = True

Effect: Bot stops asking for phone. repeated_phone_request warnings. Real lead qualification broken.

Fix: Only set phone_collected=True when a valid Nepal mobile (98/97xxxxxxxx) is extracted. Telegram user_id ≠ phone.

BUG-2 P0 — "hi" Substring False Greeting

Log: INTENT_DETECTED intent=greeting on messages containing Nepali words with "hi" inside. 
// intent_engine.py — detect_intent()
if any(word in t for word in ["hello", "hi", "namaste", ...]):
    return "greeting"

// False positives:
"hi" in "chahiyo"   → True  (500 Mbps chahiyo)
"hi" in "mahinako"  → True  (3 Mahinako kati ho?)
"hi" in "bujhina"   → True  (bujhina maile)
"hi" in "thiyo"     → True  (jodnu thiyo)

Effect: Pricing, buying, correction all routed to turn=greeting with suppress_catalog=True → validator fails → generic fallback.

Fix: Use word-boundary regex: r"\b(?:hello|hi|hey|namaste|namaskar)\b"

BUG-3 P1 — Incomplete Buying Intent Patterns

Log: Clear purchase intent not detected as buying/sales. 
Missing patterns in intent_engine.py / lead_extractor.py:
  "jodnu thiyo"        → jodn[u]? chh?[au] only (not thiyo)
  "500 Mbps chahiyo"   → no chahiyo pattern
  "100 Mbps rakhchhu"  → no rakhchhu/linu pattern
  "net jodnu chha kya" → partial match only
Fix: Add patterns: chahiy[oō], thiyo, rakhchhu, \d+\s*mbps, jodnu\s+chha

BUG-4 P0 — Validator Rejects Valid Direct Answers

Log: RESPONSE_VALIDATED is_valid=False issues=['catalog_in_non_sales_turn'] on company_info, correction, general_knowledge turns. 
// turn_router.py — _ai_identity_answer()
"Ma {name} ko AI employee hu. Tapailai internet package, coverage..."
