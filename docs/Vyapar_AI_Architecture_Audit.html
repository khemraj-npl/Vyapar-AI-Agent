1. System Under Review

Telegram-based AI employee for Nepali ISP/business use. Components: Intent Engine, Lead Engine, Memory, Knowledge Base, Product Search, Company Profile, Prompt Builder, OpenAI, CRM/leads, Render deployment.

Production logs confirm working: LEAD_SIGNALS_DETECTED, LEAD_UPSERTED, EMPLOYEE_REPLY_GENERATED (and SALES_MODE_ACTIVE after Phase 2 fixes when deployed).

User-visible failures: inconsistent memory, persona drift (sales / FAQ / Wikipedia / memory bot), weak sales conversion, pitch repetition, conversation repair failures, name corruption.

12. What Must Be Redesigned

  • LLM invocation (multi-turn, minimal prompts)
  • Output validation before Telegram send
  • Persona/phase routing (not prompt-only)
CTO one-liner: The bot can store a lead and store a name, but it cannot reason about what kind of turn this is — correction vs identity vs product vs education — so it corrupts memory and repeats templates. Do not add more regex or prompt text until Conversation State + Memory Validator + Response Validator exist.
