4. Lead Stages & Scoring

Score factors (0–100, additive)

FactorPoints
Buying intent+25
Speed mentioned+15
Location mentioned+15
Budget mentioned+10
Urgency high / medium+10 / +5
Phone / WhatsApp+20 / +18
Name, telegram contact, coverage flag+5 each

Stage rules (hard gates)

StageRequirements
newMinimal — buying only
interestedBuying + signals OR score≥40 — NO qualified without contact
qualified(phone OR whatsapp) AND (location OR speed)
hotqualified + high urgency OR medium urgency + score≥75

Stage never downgrades on re-upsert.

5. Example Flow (Consistent)

Message 1: "Baluwatar ma 500 mbps internet jodnu cha"
  buying_intent = true
  coverage_check_needed = true
  location = Baluwatar, speed = 500 Mbps
  contact = telegram only (no phone)
  score ≈ 55
  stage = interested  ← NOT qualified (no phone/WhatsApp)

Message 2: "Mero phone 9801234567 ho"
  phone added
  stage = qualified  ← phone + location + speed

Message 3: "Chhito chha aile install garna cha"
  urgency = high
  stage = hot

6. Sales Mode — Unavailable Package

When buying intent or no exact product match:

  • Do NOT inject full package catalog
  • Inject single suggested alternative via format_alternative_product()
  • Acknowledge customer need + coverage pending if ISP
  • Ask for phone or WhatsApp if missing
  • Update last_discussed_product, last_sales_question, last_sales_reply on lead
