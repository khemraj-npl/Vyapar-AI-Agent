C. Bugs & Patches Applied

BUG-1 Intent routing
Root cause: detect_intent() treats any "kati" as pricing.
File: intent_engine.py — detect_intent, _is_general_knowledge, _is_pricing_intent
Fix: General-knowledge patterns checked first; "kati" only pricing with parchha/ho/cha or product context.
BUG-2 Sales mode on non-sales turns
Root cause: _resolve_sales_mode() did not block all non-sales turn types when lead_stage=interested.
File: ai_employee_engine.py — _resolve_sales_mode
Fix: NON_SALES_TURNS frozenset returns False immediately for greeting, general_knowledge, memory_write, correction, etc.
BUG-3 Name → 300 Mbps contamination
Root cause: Name-only turn still ran upsert_lead() and injected lead_block with prior requested_speed.
Files: turn_router.py (memory_write turn), ai_employee_engine.py (suppress_lead_context), memory_validator.py (skip package on name-only)
BUG-4 Language lock failure
Root cause: Language replaced every turn; emoji flipped Nepali → English; LLM claimed "English only" in Nepali.
File: language_lock.py — sticky lock, preference detection, stronger prompt
BUG-5 Coverage repetition
Root cause: No coverage counter; COVERAGE_SALES_HINT on every sales turn.
Files: session_state_manager.py, memory_db.py, response_validator.py, turn_router.py
BUG-6 Unknown product → package pitch
Root cause: No unknown_product route.
File: turn_router.py — UNKNOWN_PRODUCT_PATTERNS → direct "confirmed price chaina"
BUG-7 Bad validator fallback
Root cause: build_fallback_reply() returned sales text on general_knowledge validation failure.
File: response_validator.py — turn-specific non-sales fallbacks
