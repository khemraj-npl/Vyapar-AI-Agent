2. Root Causes

2.1 No objection handling

There was zero logic for discount, competitor comparison, escalation, or rejection. Once lead_stage=interested, every message stayed in sales mode with the same pitch instructions.

2.2 Product block re-injected every turn

In ai_employee_engine.py, even with sales_mode=True, the system prompt always included a fresh product_block (package name + price). The model was re-fed the pitch on every turn.

2.3 SALES_EMPLOYEE_HINT replaced intent context

In prompts.py, when sales_mode=True, the specific intent hint was discarded and replaced by generic sales instructions — including “ask for phone if missing.”

2.4 Weak anti-repeat guidance

sales_memory_to_prompt() stored only the last Q/A (300 chars). Nothing explicitly told the model not to copy its previous pitch.

2.5 Single-turn OpenAI input

openai_engine.py sends one user_prompt per turn. All context must come from the system prompt. Without objection blocks, the model defaulted to the template.

4. Fix Applied

4.1 New module: sales_objection.py

Detects four objection types and generates prompt hints:

TypeExample triggersBehavior
discountdiscount, chhut, offerAnswer discount question; no re-pitch
competitorCGNET, Wlink, sasto chhaAcknowledge comparison; cite verified strengths
escalationsenior, manager, number dinusShare official phone/toll-free from profile
rejectionnajodne, mahango, chaso chhainaRespect decline; do not ask for phone

4.2 prompts.py

  • Added ANTI_REPEAT_HINT — do not copy previous package pitch
  • compose_system_prompt() now accepts objection_block and suppress_product_pitch
  • When objection detected, product pitch suppressed in system prompt

4.3 ai_employee_engine.py

  • Wire detect_sales_objection() before prompt build
  • Suppress product_block when objection detected
  • Log SALES_OBJECTION_DETECTED
  • Objection-specific rules in user prompt

4.4 lead_manager.py

Stronger anti-repeat instruction in sales_memory_to_prompt().

4.5 tests/test_sales_objection.py

Unit tests for all four objection types from production examples.
