3. Root Cause Analysis (Remaining Failures)

RC-1: Language lock wiped before DB save
resolve_session_language() sets language_locked=True in memory.
sync_session_state() reloads from DB where language_locked=False → lock lost.
Effect: "Okay", "Hello", emoji flip Nepali ↔ English every turn.
Files: language_lock.py, session_state_manager.py, ai_employee_engine.py
RC-2: Missing turn routes for follow-up and meta
"Dinus na ta" (affirmative after bot offer) and "Kasto kaaryakram chha?" (role question) fall through to general_knowledge → build_fallback_reply() spam.
Files: turn_router.py, response_validator.py
RC-3: Objection fallback ignores session language
English session + unknown product (500 Mbps) → validator returns Nepali objection text.
File: response_validator.py — build_fallback_reply()
RC-4: Meta language handler contradicts output
Bot outputs Romanized Nepali (Latin script) but claims it cannot use "English letters".
File: turn_router.py — _meta_answer() needs script/language distinction.
