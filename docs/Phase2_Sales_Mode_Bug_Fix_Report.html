1. Problem Summary

Phase 2 lead detection and persistence worked correctly in production, but sales mode never activated. The bot replied like a FAQ/general assistant instead of a sales employee.

  • Mero phone number dinchhu
  • Chhito install garna parcha

2. Root Cause

Primary: _resolve_sales_mode() in ai_employee_engine.py was too narrow. It ignored lead stage, lead score, and active lead context.

Production logs showed the mismatch clearly:

Log fieldProduction valueWhy sales_mode stayed False
buyingFalseNepali chha not matched (patterns used cha only)
stageinterestedNot used in sales_mode logic
score40–50Not used in sales_mode logic
detected_intentoften generalBlocked the intent-based path
SALES_MODE_ACTIVEnever loggedOnly logged inside sales_mode and exact_product is None

Old logic only returned True when:

  1. bundle.buying_intent == True, OR
  2. detected_intent ∈ {buying_intent, sales, pricing, coverage_inquiry} AND (speed mismatch OR coverage)

So should_process_lead() saved leads, but prompts still used full catalog FAQ mode.

Secondary: Nepali pattern gaps

User messagePattern issue
internet jodnu chhaPattern expected cha, not chha
install garna parchaNot in buying intent patterns
number dinchhuPattern was dinchu only
Kolhabi ma HONS ko net chha?Not detected as coverage inquiry

3. Fix Applied

3.1 ai_employee_engine.py — sales mode logic

_resolve_sales_mode() now also activates when:

  • lead.stage is interested, qualified, or hot
  • lead_score >= 40 (max of persisted lead + current bundle)
  • coverage_check_needed
  • should_process_lead + unavailable speed or urgency signal

Returns (bool, reason) and always logs SALES_MODE_ACTIVE when true:

SALES_MODE_ACTIVE user_id=... company_id=hons reason=lead_stage=interested stage=interested score=50

3.2 lead_extractor.py — pattern fixes

  • Buying intent: chh?[au] variants (chha/chu/cha)
  • Buying intent: install garna parcha
  • Contact: number dinchhu
  • Coverage: ko net chha, net chha?, location + ? questions

3.3 intent_engine.py — aligned patterns

Buying and coverage intent detection updated to match Nepali Telegram usage.

3.4 tests/test_sales_mode.py — new tests

  • chha variant buying intent
  • install parcha buying intent
  • Interested lead stage activates sales mode
  • Score >= 40 activates sales mode
  • Coverage question activates sales mode
